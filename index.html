하지만 쇼핑몰 프로젝트는 관리자와 사용자 페이지가 나뉘게 된다면 보안, 기능 및 권한 제한, 사용자 경험, 유지 보수 및 확장성 측면에서 많은 이점을 얻을 수 있게 되어 팀장인 저는 관리자 페이지를 담당하기로 하였습니다. 		Language Java, HTML5, CSS3, JavaScript
Tech JQuery, Ajax
Database MariaDB
FrameWork Spring, MyBatis, Bootstrap
TOOL Eclipse, HeidiSQL, VSCode



Github
사용기술
  1. AOP
    • 사용한 이유
      - 컨트롤러에서 메서드를 실행하기 전에 관리자 외 사용자의 접근을 막기 위한 조건 검사를 코드 중복을 피하기 위해
      - AOP를 활용하여 해당 조건 검사를 모듈화한 Advice를 Aspect로 분리하여 핵심 로직과 분리함
  2. Jquery(Ajax)
    • 사용한 이유
      - 중복된 데이터 입력을 실시간으로 확인하여 방지하기 위해
      - 특정 영역의 페이징 처리 및 로드를 동적으로 구현하기 위해
      - 주문 및 환불 상태 변경과 같은 작업을 비동기적으로 처리하여 페이지 전체를 다시 로드하지 않기 위해
담당 업무
담당 업무 : DB설계, 관리자 페이지(전체), 사용자 페이지(고객센터, 챌린지 이벤트)

  1. DB설계
    • 외래키 제약조건
      - 각 테이블 간의 연관성을 고려하여 외래 키 관계와 제약 조건(연쇄 삭제, 변경, No 액션)을 설정
    • View
      - 조인한 결과를 view로 생성하여 쿼리문 간소화
  2. 관리자 페이지
    • 메인 페이지(Ajax)
      - 오늘 날짜를 기준으로 주문 상태에 따른 건 수를 가져와서 JSON 객체로 반환하여 표시함
      - 최근 7일간의 매출 금액, 취소 금액, 환불 금액 및 해당 건 수를 가져와서 JSON 객체로 반환 후, 차트와 테이블로 표시함
    • 페이징 및 검색
      - 페이징 알고리즘을 사용 (이전 : -1, 다음 : +5)
      - 검색 기능과 필터링은 마이바티스 동적 쿼리를 사용
    • (다중) 이미지 파일 업로드
      - Multipart 리스트로 다중 파일 이미지를 받아와서, 반복문을 돌며 서버 경로에 각각의 파일을 업로드
      - DTO Setter 메서드에서 파일명들을 콤마(,)를 구분자로 하여 문자열로 저장
      - CK에디터를 사용하여 상세설명 이미지 업로드
    • @Transactional 어노테이션을 사용한 트랜잭션 처리 (Ajax)
      - 포인트 지급 : 1. 회원 포인트 update 2. 지급 내역 insert 3. 지급 상태 update
      - 입고 수정 : 1. 제품 수량 update 2. 입고 내역 update
    • 메일
      - mail.properties 파일과 javax.mail, javax.activation 라이브러리를 활용하여 문의 답변 시 이메일을 보냄
    • 날짜 범위검색을 통한 매출 파악
      - 리스트에서 주문 상태와 환불 상태 금액의 합을 구하고, 범위 내의 일수로 나누어 평균을 계산함
    • FAQ게시판 (Ajax)
      - RESTful API를 구현하여 CRUD 기능을 하나의 .jsp 파일에 Ajax를 사용하여 처리

팀원과의 협의 후 역할 분담을 정확하게 나누고 진행하였습니다. 		Language Java, HTML5, CSS3, JavaScript

담당 업무
담당 업무 : 형상관리, db설계, 전체 페이지 디자인, 게시판, 댓글, 발매정보/발매 소식

  1. 디자인
    • BootStrap5
      - 부트스트랩5 문서를 참고하여 시맨틱웹을 지향해 디자인 하였습니다.
    • Jquery
      - jquery를 이용해 여러 요소를 동적으로 구현하였습니다. (이미지 슬라이드, 페이지 맨 위로, 이미지 다음/이전 버튼, 대댓글폼 on/off, 쪽지함 이동 버튼 등)
  2. DB설계
    • Maria DB
      - 각 테이블간의 연관성을 고려하여 외래키관계와 제약조건(연쇄삭제, 변경)을 설정하였습니다.
  3. 게시판
    • 게시글 리스트
      - 회원 테이블과 댓글 테이블을 조인하여 작성자에 프로필사진과 게시글 제목에 댓글 수를 표시하였습니다.
      - jstl 조건문을 사용하여 제목이 길었을 때, 사진 첨부 여부, 인기 게시물 여부, 오늘 작성한 글인지 체크하여 그에 맞는 UI를 구현하였습니다.
      - 후기게시판의 경우 컨트롤러에서 img태그의 src값을 정규식으로 추출하여 썸네일로 이용하였습니다. 사진 개수도 체크하여 게시글의 몇 장의 사진이 있는지도 표기하였습니다.
    • MyBatis 동적쿼리를 이용해 게시글 필터링
      - 인기 글과 전체 글을 구분하여 볼 수 있게 구현하였습니다.
      - 제목+내용 또는 작성자로 검색할 수 있게 구현하였습니다.
      - 페이지 번호를 파라미터로 넘겨서 페이지에 맞는 게시글들이 나오도록 구현하였습니다.
    • 게시글 공개/비공개
      - 게시글 각각의 체크박스를 만들고, jstl c:set 변수에 게시글 번호를 반복문을 돌면서 추가하여 체크된 게시물 번호만 컨트롤러에서 넘겨 db에서 공개 여부 값을 참으로 만들어 처리하였습니다.
    • 게시글 추천
      - 회원만 추천 가능하게 조건설정을 하였고, 추천 버튼을 누르면 ajax를 이용하여 게시물 번호와 회원의 닉네임을 파라미터로 받아 추천 테이블로 insert 하였고, board 테이블의 추천 수는 1이 오르도록 update 하였습니다.
  4. 댓글
    • 댓글 작성폼/대댓글 작성폼
      - 회원만 작성 가능하며, 현재 로그인한 회원의 닉네임과 프로필 사진이 댓글 작성폼에 보이게 구현하였습니다.
      - 댓글마다 댓글 작성 폼을 만들어 숨겨놓고 대 댓글 작성 버튼을 누르면 해당 폼이 밑으로 보이게 구현하였습니다.
    • 댓글작성/댓글추천
      - 댓글작성과 추천시 ajax를 페이지번호와 기타 파라미터값을 받아서 db에 등록 후 페이지 이동없이 본인이 작성한 위치에서 바로 보이게 구현하였습니다.
  5. 발매정보/발매소식
    • jsoup 크롤링
      - '럭키드로우'라는 실시간 발매정보 홈페이지를 크롤링하여 이용자에게 필요한 정보의 태그와 클래스 명을 크롤링하여 객체에 저장하고 모델을 만들어 view쪽으로 넘겨서 구현하였습니다.
    • 검색
      - 크롤링하는 과정에서 String 참조변수의 contains 메서드를 이용하여 제목과 발매처를 파라미터로 받아 필터링하였습니다.
